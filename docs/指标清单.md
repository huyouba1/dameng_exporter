# 达梦 Exporter 内置指标清单

> 适用版本：dameng_exporter `v1.2.5_20260305`  
> 范围：仅整理 `/metrics` 中的**内置指标**（collector 固定暴露）；不包含 `custom_queries.metrics` 自定义指标

## 说明与约定
- **标签说明**：表中“标签”仅列采集器自身定义的标签。实际输出还会自动注入 `datasource` 以及配置中的 `labels`（如 `env/region/cluster` 等）。
- **启用开关**：
  - 数据库指标：`registerDatabaseMetrics = true`（默认 true）。
  - 主机进程指标：`registerHostMetrics = true` 且 **仅 Linux**。
  - 慢 SQL 指标：`checkSlowSQL = true` 才会采集。
  - DMHS 指标：当前未实现（`registerDmhsMetrics` 无实际指标输出）。
- **时间与单位**：
  - `*_seconds` 指标为 Unix 秒。
  - 大多数空间/内存类指标单位来自系统视图返回值（请结合 SQL 与视图含义解读）。
- **SQL 提示**：多数 SQL 使用 `/*+DM_EXPORTER*/` 或 `/*+DMDB_CHECK_FLAG*/` 提示优化。

## 6. 表空间与数据文件

### 6.1 表空间使用（TableSpaceInfoCollector）
**SQL**：
```sql
SELECT /*+DAMENG_EXPORTER*/ B.TABLESPACE_NAME
     , B.FILES
     , ROUND(B.BYTES) ASSIGNED_TOTAL_MB
     , ROUND(NVL(A.FREE_SPACE,0)) ASSIGNED_FREE_MB
     , ROUND((B.BYTES-NVL(A.FREE_SPACE,0))/B.BYTES*100,2) "USEDSZ_RT%"
     , ROUND(B.EXTENSIBLE_SPACE) EXTENS_MB
     , ROUND((NVL(A.FREE_SPACE,0)+B.EXTENSIBLE_SPACE)/(B.BYTES+B.EXTENSIBLE_SPACE)*100,2) "TOTAL_FREE%"
     , ROUND((B.BYTES-NVL(A.FREE_SPACE,0))/(B.BYTES+B.EXTENSIBLE_SPACE)*100,2) "ACTUSED_RT%"
  FROM (SELECT TABLESPACE_NAME,SUM(BYTES)/1024/1024 FREE_SPACE
          FROM DBA_FREE_SPACE GROUP BY TABLESPACE_NAME) A,
       (SELECT TABLESPACE_NAME
             ,COUNT(TABLESPACE_NAME) FILES
             ,SUM(DECODE(AUTOEXTENSIBLE,'NO',0,DECODE(SIGN(MAXBYTES-BYTES),1,MAXBYTES-BYTES,0)))/1024/1024 EXTENSIBLE_SPACE
             ,SUM(BYTES)/1024/1024 BYTES
          FROM DBA_DATA_FILES
         GROUP BY TABLESPACE_NAME) B
 WHERE A.TABLESPACE_NAME(+)=B.TABLESPACE_NAME
 ORDER BY 7
```

| 指标名 | 类型 | 标签 | 指标含义 | 注意事项/取值说明 |
| --- | --- | --- | --- | --- |
| `dmdbms_tablespace_size_total_info` | gauge | `tablespace_name` | 表空间分配总量（MB） | 使用结果列 `ASSIGNED_TOTAL_MB` |
| `dmdbms_tablespace_size_free_info` | gauge | `tablespace_name` | 表空间分配空闲（MB） | 使用结果列 `ASSIGNED_FREE_MB` |
| `dmdbms_tablespace_size_act_used_rt_info` | gauge | `tablespace_name` | 表空间实际使用率（%） | 使用结果列 `ACTUSED_RT%`，取值范围通常为 `0-100` |

注意事项：采集结果会缓存（`bigKeyDataCacheTime`）。

## 7. 内存池与缓冲池

### 7.1 内存池（DbMemoryPoolInfoCollector）
**SQL**：
```sql
WITH stat_values AS (
    SELECT 
        SUM(CASE WHEN ID IN (114, 115) THEN STAT_VAL END) AS hj_curr_val,
        MAX(CASE WHEN ID = 145 THEN STAT_VAL END) AS hj_res_val,
        MAX(CASE WHEN ID = 144 THEN STAT_VAL END) AS hj_total_val,
        SUM(CASE WHEN ID = 116 THEN STAT_VAL END) AS hagr_curr_val,
        MAX(CASE WHEN ID = 143 THEN STAT_VAL END) AS hagr_res_val,
        MAX(CASE WHEN ID = 142 THEN STAT_VAL END) AS hagr_total_val,
        SUM(CASE WHEN ID = 178 THEN STAT_VAL END) AS sort_curr_val,
        MAX(CASE WHEN ID = 177 THEN STAT_VAL END) AS sort_total_val
    FROM V$SYSSTAT
    WHERE ID IN (114, 115, 116, 142, 143, 144, 145, 177, 178)
)
SELECT /*+DM_EXPORTER*/ ZONE_TYPE, CURR_VAL, RES_VAL, TOTAL_VAL
FROM (
    SELECT 'HJ ZONE' AS ZONE_TYPE, hj_curr_val AS CURR_VAL, hj_res_val AS RES_VAL, hj_total_val AS TOTAL_VAL FROM stat_values
    UNION ALL
    SELECT 'HAGR ZONE' AS ZONE_TYPE, hagr_curr_val AS CURR_VAL, hagr_res_val AS RES_VAL, hagr_total_val AS TOTAL_VAL FROM stat_values
    UNION ALL
    SELECT 'SORT ZONE' AS ZONE_TYPE, sort_curr_val AS CURR_VAL, NULL AS RES_VAL, sort_total_val AS TOTAL_VAL FROM stat_values
)
```

| 指标名 | 类型 | 标签 | 指标含义 | 注意事项/取值说明 |
| --- | --- | --- | --- | --- |
| `dmdbms_memory_total_pool_info` | gauge | `pool_type` | 内存池总量 | `pool_type` 取值：`HJ ZONE` / `HAGR ZONE` / `SORT ZONE` |
| `dmdbms_memory_curr_pool_info` | gauge | `pool_type` | 内存池当前使用量 | 单位与 `V$SYSSTAT.STAT_VAL` 一致 |

### 8.4 定时作业错误数（DbJobRunningInfoCollector）
**SQL**：
```sql
SELECT /*+DM_EXPORTER*/ COUNT(*) error_num
FROM (
  SELECT NAME, ERRINFO
  FROM SYSJOB.SYSJOBHISTORIES2
  WHERE ERRCODE != 0
    AND START_TIME >= (SYSDATE-31)
    AND NAME IN (
      SELECT SYSJOBS.NAME
      FROM SYSJOB.SYSJOBSCHEDULES SCHE
      LEFT JOIN SYSJOB.USER_JOBS USERJOB ON SCHE.JOBID = USERJOB.JOB
      LEFT JOIN SYSJOB.SYSJOBSTEPS STEPS ON SCHE.JOBID = STEPS.JOBID
      LEFT JOIN SYSJOB.SYSJOBS SYSJOBS ON SCHE.JOBID = SYSJOBS.ID
      WHERE STEPS."TYPE" = 6 AND SCHE.VALID = 'Y'
      GROUP BY SYSJOBS.NAME
    )
  UNION ALL
  SELECT NAME, ERRINFO
  FROM SYSJOB.SYSSTEPHISTORIES2
  WHERE ERRCODE != 0
    AND START_TIME >= (SYSDATE-31)
    AND NAME IN (
      SELECT SYSJOBS.NAME
      FROM SYSJOB.SYSJOBSCHEDULES SCHE
      LEFT JOIN SYSJOB.USER_JOBS USERJOB ON SCHE.JOBID = USERJOB.JOB
      LEFT JOIN SYSJOB.SYSJOBSTEPS STEPS ON SCHE.JOBID = STEPS.JOBID
      LEFT JOIN SYSJOB.SYSJOBS SYSJOBS ON SCHE.JOBID = SYSJOBS.ID
      WHERE STEPS."TYPE" = 6 AND SCHE.VALID = 'Y'
      GROUP BY SYSJOBS.NAME
    )
)
```

| 指标名 | 类型 | 标签 | 指标含义 | 注意事项/取值说明 |
| --- | --- | --- | --- | --- |
| `dmdbms_joblog_error_num` | gauge | （无） | 近 31 天定时作业错误数量 | 若未启用作业系统会直接跳过并提示执行 `SP_INIT_JOB_SYS(1)` |

### 10.3 归档发送详情/差值（DbArchSendCollector）
**SQL（根据视图存在性选择）**：
```sql
-- 如果存在 V$ARCH_APPLY_INFO：
SELECT /*+DM_EXPORTER*/
  ARCH_DEST, ARCH_TYPE,
  CASE WHEN EXISTS (
    SELECT 1 FROM V$ARCH_APPLY_INFO
    WHERE V$ARCH_SEND_INFO.ARCH_DEST = V$ARCH_APPLY_INFO.DEST_NAME
  ) THEN MAX_SEND_LSN - (
    SELECT RPKG_LSN FROM V$ARCH_APPLY_INFO
    WHERE V$ARCH_SEND_INFO.ARCH_DEST = V$ARCH_APPLY_INFO.DEST_NAME
  ) ELSE 0 END AS LSN_DIFFERENCE,
  LAST_SEND_CODE, LAST_SEND_DESC,
  TO_CHAR(LAST_START_TIME,'YYYY-MM-DD HH24:MI:SS') AS LAST_START_TIME,
  TO_CHAR(LAST_END_TIME,'YYYY-MM-DD HH24:MI:SS') AS LAST_END_TIME,
  LAST_SEND_TIME
FROM V$ARCH_SEND_INFO;

-- 否则：
SELECT /*+DM_EXPORTER*/
  ARCH_DEST, ARCH_TYPE,
  (MAX_SEND_LSN - LAST_SEND_LSN) AS LSN_DIFFERENCE,
  LAST_SEND_CODE, LAST_SEND_DESC,
  TO_CHAR(LAST_START_TIME,'YYYY-MM-DD HH24:MI:SS') AS LAST_START_TIME,
  TO_CHAR(LAST_END_TIME,'YYYY-MM-DD HH24:MI:SS') AS LAST_END_TIME,
  LAST_SEND_TIME
FROM V$ARCH_SEND_INFO;
```

| 指标名 | 类型 | 标签 | 指标含义 | 注意事项/取值说明 |
| --- | --- | --- | --- | --- |
| `dmdbms_arch_send_detail_info` | gauge | `arch_type, arch_dest` | 归档发送 LSN 差值 | 值为 `LSN_DIFFERENCE` |
| `dmdbms_arch_send_diff_value` | gauge | `arch_type, arch_dest` | 归档发送 LSN 差值（简化指标） | 与上同值，便于告警/图表 |

注意事项：归档未启用时不输出；若 `V$ARCH_SEND_INFO` 缺少 `LAST_SEND_CODE/LAST_SEND_DESC` 字段，会自动替换为空字符串。
