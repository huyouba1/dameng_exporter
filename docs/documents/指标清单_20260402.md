# 达梦 Exporter 内置指标清单

> 适用版本：Dameng_exporter `v1.2.5_20260305`  
> 范围：整理 `/metrics` 中的**内置指标**（collector 固定暴露）

## 说明与约定

- **标签说明**：表中的“标签”仅列采集器自身定义的标签。实际输出时还会自动注入 `datasource` 以及配置中的 `labels`（如 `env/region/cluster`）。
- **启用开关**：
  - 数据库指标：`registerDatabaseMetrics = true`（默认开启）。
  - 主机进程指标：`registerHostMetrics = true` 且 **仅 Linux**。

## 5. 实例运行状态（DBInstanceRunningInfoCollector）

**SQL**：

## 6. 表空间与数据文件

### 6.1 表空间使用（TableSpaceInfoCollector）

**SQL**：
```sql
SELECT /*+DAMENG_EXPORTER*/
       B.TABLESPACE_NAME,
       B.FILES,
       ROUND(B.BYTES) ASSIGNED_TOTAL_MB,
       ROUND(NVL(A.FREE_SPACE,0)) ASSIGNED_FREE_MB,
       ROUND((B.BYTES-NVL(A.FREE_SPACE,0))/B.BYTES*100,2) "USEDSZ_RT%",
       ROUND(B.EXTENSIBLE_SPACE) EXTENS_MB,
       ROUND((NVL(A.FREE_SPACE,0)+B.EXTENSIBLE_SPACE)/(B.BYTES+B.EXTENSIBLE_SPACE)*100,2) "TOTAL_FREE%",
       ROUND((B.BYTES-NVL(A.FREE_SPACE,0))/(B.BYTES+B.EXTENSIBLE_SPACE)*100,2) "ACTUSED_RT%"
FROM (
    SELECT TABLESPACE_NAME, SUM(BYTES)/1024/1024 FREE_SPACE
    FROM DBA_FREE_SPACE
    GROUP BY TABLESPACE_NAME
) A,
(
    SELECT TABLESPACE_NAME,
           COUNT(TABLESPACE_NAME) FILES,
           SUM(DECODE(AUTOEXTENSIBLE,'NO',0,DECODE(SIGN(MAXBYTES-BYTES),1,MAXBYTES-BYTES,0)))/1024/1024 EXTENSIBLE_SPACE,
           SUM(BYTES)/1024/1024 BYTES
    FROM DBA_DATA_FILES
    GROUP BY TABLESPACE_NAME
) B
WHERE A.TABLESPACE_NAME(+)=B.TABLESPACE_NAME
ORDER BY 7;
```

| 指标名 | 类型 | 标签 | 对应 SQL 列 / 逻辑 | 说明与注意事项 |
| --- | --- | --- | --- | --- |
| `dmdbms_tablespace_size_total_info` | gauge | `tablespace_name` | `ASSIGNED_TOTAL_MB` | 表空间总分配大小。 |
| `dmdbms_tablespace_size_free_info` | gauge | `tablespace_name` | `ASSIGNED_FREE_MB` | 表空间剩余大小。 |
| `dmdbms_tablespace_size_act_used_rt_info` | gauge | `tablespace_name` | `ACTUSED_RT%` | 表空间实际使用率。 |

**注意事项**：
- `FILES`、`USEDSZ_RT%`、`EXTENS_MB`、`TOTAL_FREE%` 在 SQL 中返回，但当前没有单独暴露为 Prometheus 指标。

## 7. 内存池与缓冲池

### 7.1 内存池（DbMemoryPoolInfoCollector）

**SQL**：
```sql
WITH stat_values AS (
    SELECT
        SUM(CASE WHEN ID IN (114, 115) THEN STAT_VAL END) AS hj_curr_val,
        MAX(CASE WHEN ID = 145 THEN STAT_VAL END) AS hj_res_val,
        MAX(CASE WHEN ID = 144 THEN STAT_VAL END) AS hj_total_val,
        SUM(CASE WHEN ID = 116 THEN STAT_VAL END) AS hagr_curr_val,
        MAX(CASE WHEN ID = 143 THEN STAT_VAL END) AS hagr_res_val,
        MAX(CASE WHEN ID = 142 THEN STAT_VAL END) AS hagr_total_val,
        SUM(CASE WHEN ID = 178 THEN STAT_VAL END) AS sort_curr_val,
        MAX(CASE WHEN ID = 177 THEN STAT_VAL END) AS sort_total_val
    FROM V$SYSSTAT
    WHERE ID IN (114, 115, 116, 142, 143, 144, 145, 177, 178)
)
SELECT /*+DM_EXPORTER*/ ZONE_TYPE, CURR_VAL, RES_VAL, TOTAL_VAL
FROM (
    SELECT 'HJ ZONE' AS ZONE_TYPE, hj_curr_val AS CURR_VAL, hj_res_val AS RES_VAL, hj_total_val AS TOTAL_VAL FROM stat_values
    UNION ALL
    SELECT 'HAGR ZONE' AS ZONE_TYPE, hagr_curr_val AS CURR_VAL, hagr_res_val AS RES_VAL, hagr_total_val AS TOTAL_VAL FROM stat_values
    UNION ALL
    SELECT 'SORT ZONE' AS ZONE_TYPE, sort_curr_val AS CURR_VAL, NULL AS RES_VAL, sort_total_val AS TOTAL_VAL FROM stat_values
);
```

| 指标名 | 类型 | 标签 | 对应 SQL 列 / 逻辑 | 说明与注意事项 |
| --- | --- | --- | --- | --- |
| `dmdbms_memory_total_pool_info` | gauge | `pool_type` | `TOTAL_VAL` | 按 `pool_type` 区分 `HJ ZONE`、`HAGR ZONE`、`SORT ZONE`。 |
| `dmdbms_memory_curr_pool_info` | gauge | `pool_type` | `CURR_VAL` | 当前已使用的内存池大小。 |

**注意事项**：
- `RES_VAL` 在 SQL 中返回，但当前没有单独暴露为指标。

### 8.4 定时作业错误数（DbJobRunningInfoCollector）

**SQL**：
```sql
SELECT /*+DM_EXPORTER*/ COUNT(*) error_num
FROM (
    SELECT NAME, ERRINFO
    FROM SYSJOB.SYSJOBHISTORIES2
    WHERE ERRCODE != 0
      AND START_TIME >= (SYSDATE - 31)
      AND NAME IN (
          SELECT SYSJOBS.NAME
          FROM SYSJOB.SYSJOBSCHEDULES SCHE
          LEFT JOIN SYSJOB.USER_JOBS USERJOB ON SCHE.JOBID = USERJOB.JOB
          LEFT JOIN SYSJOB.SYSJOBSTEPS STEPS ON SCHE.JOBID = STEPS.JOBID
          LEFT JOIN SYSJOB.SYSJOBS SYSJOBS ON SCHE.JOBID = SYSJOBS.ID
          WHERE STEPS."TYPE" = 6 AND SCHE.VALID = 'Y'
          GROUP BY SYSJOBS.NAME
      )
    UNION ALL
    SELECT NAME, ERRINFO
    FROM SYSJOB.SYSSTEPHISTORIES2
    WHERE ERRCODE != 0
      AND START_TIME >= (SYSDATE - 31)
      AND NAME IN (
          SELECT SYSJOBS.NAME
          FROM SYSJOB.SYSJOBSCHEDULES SCHE
          LEFT JOIN SYSJOB.USER_JOBS USERJOB ON SCHE.JOBID = USERJOB.JOB
          LEFT JOIN SYSJOB.SYSJOBSTEPS STEPS ON SCHE.JOBID = STEPS.JOBID
          LEFT JOIN SYSJOB.SYSJOBS SYSJOBS ON SCHE.JOBID = SYSJOBS.ID
          WHERE STEPS."TYPE" = 6 AND SCHE.VALID = 'Y'
          GROUP BY SYSJOBS.NAME
      )
);
```

| 指标名 | 类型 | 标签 | 对应 SQL 列 / 逻辑 | 说明与注意事项 |
| --- | --- | --- | --- | --- |
| `dmdbms_joblog_error_num` | gauge | 无 | `ERROR_NUM` | 统计最近 31 天相关作业执行错误数。 |

### 9.6 DW 守护状态（DbDwWatcherInfoCollector）

**SQL**：
```sql
SELECT /*+DMDB_CHECK_FLAG*/
       WATCHER.DW_MODE,
       WATCHER.DW_STATUS,
       WATCHER.AUTO_RESTART,
       CASE WATCHER.DW_STATUS
            WHEN 'OPEN' THEN '1'
            WHEN 'MOUNT' THEN '2'
            WHEN 'SUSPEND' THEN '3'
            ELSE '4'
       END AS DW_STATUS_TO_NUM
FROM V$DMWATCHER WATCHER
```

### 10.1 归档状态（DbArchStatusCollector）

**SQL**：
```sql
-- 本地归档总状态探测
SELECT /*+DMDB_CHECK_FLAG*/ PARA_VALUE FROM V$DM_INI WHERE PARA_NAME = 'ARCH_INI';

SELECT /*+DMDB_CHECK_FLAG*/
       CASE ARCH_STATUS
            WHEN 'VALID' THEN 'VALID'
            WHEN 'INVALID' THEN 'INVALID'
            WHEN 'ASYNC_SEND' THEN 'ASYNC_SEND'
            ELSE 'UNKNOWN'
       END AS ARCH_STATUS
FROM V$ARCH_STATUS
WHERE ARCH_TYPE = 'LOCAL';

       CASE
            WHEN ARCH_STATUS = 'INVALID' THEN 0
            WHEN ARCH_STATUS = 'VALID' THEN 1
            WHEN ARCH_STATUS = 'ASYNC_SEND' THEN 2
            WHEN ARCH_STATUS IS NULL OR TRIM(ARCH_STATUS) = '' THEN -3
            ELSE -2
       END AS ARCH_STATUS,
       ARCH_TYPE,
       ARCH_DEST,
       ARCH_SRC
FROM V$ARCH_STATUS;
```

### 10.3 归档发送详情 / 差值（DbArchSendCollector）

**SQL**：
```sql
-- 归档是否开启
SELECT /*+DMDB_CHECK_FLAG*/ PARA_VALUE FROM V$DM_INI WHERE PARA_NAME = 'ARCH_INI';

-- 字段探测
SELECT /*+DM_EXPORTER*/ COUNT(*)
FROM V$DYNAMIC_TABLE_COLUMNS
WHERE TABNAME = 'V$ARCH_SEND_INFO'
  AND COLNAME IN ('LAST_SEND_CODE','LAST_SEND_DESC');

-- 视图探测
SELECT /*+DM_EXPORTER*/ COUNT(1)
FROM V$DYNAMIC_TABLES
WHERE NAME = 'V$ARCH_APPLY_INFO';

-- 分支 1：无 V$ARCH_APPLY_INFO 时
SELECT /*+DM_EXPORTER*/
       ARCH_DEST,
       ARCH_TYPE,
       (MAX_SEND_LSN - LAST_SEND_LSN) AS LSN_DIFFERENCE,
       LAST_SEND_CODE,
       LAST_SEND_DESC,
       TO_CHAR(LAST_START_TIME,'YYYY-MM-DD HH24:MI:SS') AS LAST_START_TIME,
       TO_CHAR(LAST_END_TIME,'YYYY-MM-DD HH24:MI:SS') AS LAST_END_TIME,
       LAST_SEND_TIME
FROM V$ARCH_SEND_INFO;

-- 分支 2：存在 V$ARCH_APPLY_INFO 时
SELECT /*+DM_EXPORTER*/
       ARCH_DEST,
       ARCH_TYPE,
       CASE
            WHEN EXISTS (
                SELECT 1
                FROM V$ARCH_APPLY_INFO
                WHERE V$ARCH_SEND_INFO.ARCH_DEST = V$ARCH_APPLY_INFO.DEST_NAME
            )
            THEN MAX_SEND_LSN - (
                SELECT RPKG_LSN
                FROM V$ARCH_APPLY_INFO
                WHERE V$ARCH_SEND_INFO.ARCH_DEST = V$ARCH_APPLY_INFO.DEST_NAME
            )
            ELSE 0
       END AS LSN_DIFFERENCE,
       LAST_SEND_CODE,
       LAST_SEND_DESC,
       TO_CHAR(LAST_START_TIME,'YYYY-MM-DD HH24:MI:SS') AS LAST_START_TIME,
       TO_CHAR(LAST_END_TIME,'YYYY-MM-DD HH24:MI:SS') AS LAST_END_TIME,
       LAST_SEND_TIME
FROM V$ARCH_SEND_INFO;
```

| 指标名 | 类型 | 标签 | 对应 SQL 列 / 逻辑 | 说明与注意事项 |
| --- | --- | --- | --- | --- |
| `dmdbms_arch_send_detail_info` | gauge | `arch_type, arch_dest` | `LSN_DIFFERENCE` | 与 `dmdbms_arch_send_diff_value` 使用相同值。 |
| `dmdbms_arch_send_diff_value` | gauge | `arch_type, arch_dest` | `LSN_DIFFERENCE` | 用于直接查看归档发送差值。 |
| `dmdbms_arch_send_last_code` | gauge | `arch_type, arch_dest` | `LAST_SEND_CODE` | 仅当 `LAST_SEND_CODE` 字段存在且能转成数字时输出。 |

**注意事项**：

- 如果 `LAST_SEND_CODE`、`LAST_SEND_DESC` 字段不存在，代码会把 SQL 文本中的这两列替换成空字符串占位，以保证扫描顺序不变。
- `LAST_SEND_DESC`、`LAST_START_TIME`、`LAST_END_TIME`、`LAST_SEND_TIME` 会被扫描，但当前不转成 Prometheus 指标。
