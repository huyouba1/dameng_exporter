# 阻塞会话监控 一个会话锁超过1小时，且阻塞其他用户
自定义指标方式实现

1. 在dameng_exporter.toml配置的数据源中
```
[[datasource]]
name = "prod_primary_01"
description = "生产环境主库-订单系统"
enabled = true
dbHost = "192.168.112.179:5236"
dbUser = "SYSDBA"
dbPwd = "Dameng123"  # 支持ENC()加密格式
labels = "env=prod,region=beijing,cluster=order_cluster"
customMetricsFile = "./custom_queries.metrics"
```
2. 在custom_queries.metrics中新增指标

```
# ================================================================================
# 自定义: 事务等待超过1小时数量
# 说明：统计等待时间超过1小时（毫秒）的事务等待条目数
# ================================================================================
[[metric]]
context = "trxwait_over_1h"
labels = []
request = "SELECT COUNT(*) AS wait_count FROM V$TRXWAIT WHERE WAIT_TIME > 60*60*1000"
metricsdesc = { wait_count = "等待时间超过1小时的事务等待数量" }
metricstype = { wait_count = "gauge" }
```

3. 重启程序后，看该指标是否存在

```
# HELP dmdbms_trxwait_over_1h_wait_count 等待时间超过1小时的事务等待数量
# TYPE dmdbms_trxwait_over_1h_wait_count gauge
dmdbms_trxwait_over_1h_wait_count{cluster="order_cluster",datasource="prod_primary_01@192.168.112.179:5236",env="prod",region="beijing"} 0
```

4. rule文件告警配置
在docs/protheues_告警规则/rules目录下
```
  - alert: 等待时间超过1小时的事务等待数量
    expr: dmdbms_trxwait_over_1h_wait_count > 0
    for: 15s
    labels:
      severity: 警告
    annotations:
      description: "数据源：{{ $labels.datasource }} 数据库存在等待时间超过1小时的事务等待数量，当前阻塞数：{{ printf \"%.0f\" $value }}"
      summary: "数据源：{{ $labels.datasource }}:存在等待时间超过1小时的事务等待数量，当前阻塞数{{ printf \"%.0f\" $value }}，请及时处理"   
```
